Contains the main emulation classes and some common utilities.

The class EmulationSystem manages the emulation. It controls the device tree, schedules all events and allows you to run emulation for a given amount of time.

The class EmulationDevice represents a device that can run in an EmulationSystem. It must belong to an EmulationSystem to work.

The class EmulationEvent specifies an event that can occur inside a given device. Events are dispatched by the EmulationSystem which the device belongs to.

The class EmulationClock represents a special device representing a clock. Events can be attached to an EmulationClock instead of a EmulationSystem for cycle accurate emulation and easy management.

The class EmulationThread allows to run an EmulationSystem in a separate thread. It allows also to control the speed of emulation and gives performance informations.

The class EmulationTimer provides a time base for the EmulationThread. The class SystemMillisecondTimer is a subclass of EmulationTimer which uses the method System.currentTimeMillis(). This is generally the class you will use but you can also implements a more accurate timer. Note that if you implement your own EmulationTimer, you should not use the method System.nanoTime() under Windows: it is very slow, eating a lot of CPU with just a few calls and causing jerky emulation.

The class EmulationState contains the state of an EmulationSystem.

The class TimeStamp is used to represents a given time during emulation.

The class FastString is used to quickly build strings. It is mainly used for debugging.

The class Tools contains some simple common tools.
